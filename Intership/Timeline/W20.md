1. Pipeline Keypoint Map Generation
First, you should modify the keypoint map generation in pipeline.py to respect the mask:

```python
def add_keypoint_map(data):
    with tf.name_scope('add_keypoint_map'):
        image_shape = tf.shape(data['image'])[:2]
        kp = tf.minimum(tf.to_int32(tf.round(data['keypoints'])), image_shape-1)
        kmap = tf.scatter_nd(
                kp, tf.ones([tf.shape(kp)[0]], dtype=tf.int32), image_shape)
        
        # Apply mask if available to filter keypoints in invalid regions
        mask = data.get('mask', None)
        if mask is not None:
            kmap = tf.where(mask, kmap, tf.zeros_like(kmap))
            
    return {**data, 'keypoint_map': kmap}
```

This ensures keypoint maps only contain points in valid regions of the image.

3. Metrics Calculation
Update the metrics calculation in both model files:

```python
def _metrics(self, outputs, inputs, **config):
    # Get the mask - either from the combined mask or the default valid_mask
    mask = inputs.get('mask', inputs.get('valid_mask'))
    
    # Apply mask to predictions
    pred = mask * outputs['pred']
    labels = inputs['keypoint_map']

    # Calculate metrics only on valid regions
    precision = tf.reduce_sum(pred * labels) / (tf.reduce_sum(pred) + 1e-8)
    recall = tf.reduce_sum(pred * labels) / (tf.reduce_sum(labels) + 1e-8)

    return {'precision': precision, 'recall': recall}
```

```python
def filter_points(points, shape, mask=None):
    with tf.name_scope('filter_points'):
        # First filter by image boundaries
        in_bounds = (points >= 0) & (points <= tf.to_float(shape-1))
        valid_points = tf.boolean_mask(points, tf.reduce_all(in_bounds, -1))
        
        # If mask is provided, further filter by mask
        if mask is not None:
            # Convert points to indices
            indices = tf.cast(tf.round(valid_points), tf.int32)
            
            # Get mask values at point locations
            mask_values = tf.gather_nd(mask, indices)
            
            # Keep only points where mask is True
            valid_points = tf.boolean_mask(valid_points, mask_values)
            
        return valid_points
```

## Training

When you start training in this codebase, the process is orchestrated by the `BaseModel` class (see superpoint/models/base_model.py). Here’s a step-by-step explanation of what happens:

---

### 1. **Training is Started**

The training is typically started by calling the `train()` method of your model (e.g., `SuperPoint`). This is usually triggered from a script like experiment.py via the `train()` function.

_loss---

### 2. **Building the Graph**

When the model is initialized, `_build_graph()` is called. This sets up the computation graph for training, evaluation, and prediction. For training, it calls `_train_graph()`.

---

### 3. **Data Sharding and GPU Towers**

In `_train_graph()`, the method `_gpu_tower()` is called. This function:
- Splits the batch across available GPUs (or uses CPU if no GPU).
- For each shard (mini-batch for a device), it calls the model’s `_model()` method to get predictions (`net_outputs`).
- If in training mode (`Mode.TRAIN`), it then calls the model’s `_loss()` method to compute the loss for that shard.

**Relevant code:**
```python
net_outputs = self._model(shards[i], mode, **self.config)
if mode == Mode.TRAIN:
    loss = self._loss(net_outputs, shards[i], **self.config)
```

---

### 4. **Loss and Gradients**

- The `_loss()` method (implemented in your model, e.g., `SuperPoint`) receives:
  - `outputs`: the predictions from `_model()` (logits, probabilities, descriptors, etc.)
  - `inputs`: the input data for this shard (images, keypoint maps, masks, etc.)
  - `config`: configuration dictionary

- The loss is computed (including regularization losses), and gradients are calculated with respect to model parameters.

---

### 5. **Averaging and Optimization**

- Gradients from all towers are averaged.
- The optimizer (e.g., Adam) applies the gradients to update the model parameters.
- The global loss is tracked for logging and summaries.

---

### 6. **Training Loop**

- The `train()` method runs a loop for the specified number of iterations.
- In each iteration, it runs the session to compute the loss, summaries, and apply the optimizer.
- Periodically, it saves checkpoints and evaluates on the validation set.

**Relevant code:**
```python
for i in range(iterations):
    loss, summaries, _ = self.sess.run(
        [self.loss, self.summaries, self.trainer],
        feed_dict={self.handle: self.dataset_handles['training']},
        options=options, run_metadata=run_metadata)
```

---

### 7. **Summary**

- **Data** is loaded and split for each device.
- **_model()** builds the forward pass and returns predictions.
- **_loss()** computes the loss using predictions and ground truth.
- **Gradients** are computed and applied.
- **Training** proceeds for the specified number of iterations, with periodic evaluation and checkpointing.

---

**In short:**  
During training, the model’s `_model()` and `_loss()` methods are called for each batch (and each GPU if using multiple). The outputs and inputs are passed as dictionaries, and the loss is minimized using the optimizer. This is all managed by the `BaseModel` class and its training loop.

If you want to debug or inspect variables during training, you can set breakpoints in `_model()` or `_loss()` of your model class, or in the training loop itself.
