### Understanding `top_k` in SuperPoint

In your SuperPoint configuration file, `top_k` is an important parameter that controls keypoint selection:

#### What `top_k` Does

`top_k` sets the **maximum number of keypoints** that the model will output from each image. The detection process works as follows:

1. SuperPoint generates a **heatmap** across the entire image showing potential keypoint locations
2. **Non-maximum suppression** (controlled by your `nms: 4` parameter) eliminates clusters of nearby points
3. The remaining points are **sorted by confidence score** (how "interesting" or distinctive they are)
4. Only the **top K highest-scoring points** are kept in the final output

#### Why It Matters for Your Project

For your colonoscopy images, setting an appropriate `top_k` value is particularly important because:

1. **Balance quality vs. quantity**: Too many keypoints may include unreliable points from specular highlights or black edges
2. **Computational efficiency**: Fewer points means faster processing when matching features
3. **Consistent density**: Ensures similar number of keypoints across different colonoscopy sequences

#### Recommended Settings

For your colonoscopy project steps:

- **Step 1 (Initial training)**: Uncomment and set `top_k: 1000` to capture sufficient features despite distortion
- **Step 3 (With masks)**: Keep `top_k: 1000`, as your masks will naturally filter out unwanted keypoints
- **Step 4 (With distortion correction)**: You might reduce to `top_k: 800` as features will be more reliable after correction

I recommend uncommenting the line and setting an explicit value based on your specific needs.

## 2 Rounds of HA
#### Use the First Round's Output as Input for the Second Round
To perform a second round of HA:
1. Update the configuration file (`configs/magic-point_coco_export.yaml`) to point to the output of the first round:
   ```yaml
   data:
       labels: outputs/magic-point_coco-export1
   ```
   This ensures that the first round's pseudo-labels are used as input for the second round.

2. Run the export command again, but save the output to a new directory:
   ```bash
   python superpoint/export_detections.py \
     configs/magic-point_coco_export.yaml \
     magic-point_synth \
     --pred_only \
     --batch_size=5 \
     --export_name=magic-point_coco-export2
   ```

This will save the second round of pseudo-ground truth keypoints to:
```
$EXPER_PATH/outputs/magic-point_coco-export2/
```

#### Why Perform Multiple Rounds of HA?
- Each round of HA refines the pseudo-ground truth keypoints, making them more robust to geometric transformations.
- This improves the quality of the training data for SuperPoint, leading to better performance.
