## Data structure of input
Here are all the possible keys and structures for the `input` dictionary in your `Colon` dataset, with a focus on the different `valid_mask` variants you might encounter or want to update:

---

### `input` Dictionary: Possible Keys

### Always Present
- **'image'**: Main image tensor (original, augmented, or warped).
- **'name'**: Image name.
- **'scene_name'**: Dummy scene name.

---

### Masks (valid regions)
- **'valid_mask'**:  
  - Mask for the main image (original or augmented).
  - Can be a mask for the warped image if homographic augmentation is enabled.
  - Can be a mask combining camera mask and specular mask (as in your `_get_extra_mask`).
- **'warped_valid_mask'**:  
  - Mask for the warped image (when `warped_pair` is enabled).
- **'valid_mask'** (in homography adaptation):  
  - Batch of masks for each warped image (shape: `[N, H, W]`).

---

### Images
- **'image'**:  
  - Original image, augmented image, or warped image.
- **'image_2D'**:  
  - Original image before warping (used in homography adaptation).
- **'warped_img'**:  
  - Warped image (when `warped_pair` is enabled).

---

### Labels
- **'labels_2D'**:  
  - Keypoint label map for the image.
- **'labels_res'**:  
  - Residuals for keypoints.
- **'warped_labels'**:  
  - Warped label map (when `warped_pair` is enabled).
- **'warped_res'**:  
  - Warped residuals.
- **'labels_2D_gaussian'**:  
  - Gaussian-blurred label map.
- **'warped_labels_gaussian'**:  
  - Gaussian-blurred warped labels.
- **'warped_labels_bi'**:  
  - Bilinear warped labels.

---

### Homographies
- **'homographies'**:  
  - Homography matrices (for augmentation or warped pairs).
- **'inv_homographies'**:  
  - Inverse homographies.

---

## How to Update All Possible `valid_mask` Keys

You can update all masks in the `input` dictionary like this:

````python
# Update main valid_mask
input['valid_mask'] = new_valid_mask

# Update warped valid_mask if present
if 'warped_valid_mask' in input:
    input['warped_valid_mask'] = new_warped_valid_mask

# Update batch valid_mask for homography adaptation
if isinstance(input.get('valid_mask'), torch.Tensor) and input['valid_mask'].dim() == 3:
    input['valid_mask'] = new_batch_valid_mask
````

## descriptor_loss
### 🧠 High-Level Goal

Given:

- A grid of local descriptors extracted from an image.
- A homography that warps the image.
- Compute a **contrastive loss** that pulls **corresponding descriptors** close and **non-corresponding** ones apart.

---

### 🧩 1. Coordinate setup

```python
(batch_size, Hc, Wc) = tf.unstack(tf.to_int32(tf.shape(descriptors)[:3]))
```

✅ **Extract height and width** of the descriptor grid.  
If the image was 960×1280 and `grid_size=8`, then:  
**Hc = 120, Wc = 160**

---

```python
coord_cells = tf.stack(tf.meshgrid(
        tf.range(Hc), tf.range(Wc), indexing='ij'), axis=-1)
coord_cells = coord_cells * config['grid_size'] + config['grid_size'] // 2
```

✅ **Build the 2D grid** of center coordinates of each 8×8 cell:

- Shape: `(Hc, Wc, 2)`
    
- Each value is `(x, y)` center of a patch
    

🧮 You get coordinates like:

```
(0,0) → (4,4)
(0,1) → (4,12)
(1,0) → (12,4)
```

👁️ Visually: a grid of center points like this:

```
+---+---+---+
| x | x | x |
+---+---+---+
| x | x | x |
+---+---+---+
```

---

### 🌀 2. Warped coordinates

```python
warped_coord_cells = warp_points(tf.reshape(coord_cells, [-1, 2]), homographies)
```

✅ Apply homographies to every coordinate center.

🧮 Mathematically:

$$
\mathbf{p}' = H \cdot \mathbf{p}
$$

You now have:  
**`warped_coord_cells.shape = (B, Hc×Wc, 2)`**

👁️ Think of this as saying:  
“Each descriptor at (h,w) in image A is now located at some subpixel position in image B.”

---

### 📏 3. Find pairs of matching pixels

```python
coord_cells = tf.reshape(coord_cells, [1, 1, 1, Hc, Wc, 2])
warped_coord_cells = tf.reshape(warped_coord_cells, [B, Hc, Wc, 1, 1, 2])
cell_distances = tf.norm(coord_cells - warped_coord_cells, axis=-1)
```

✅ Compute **all-pairs spatial distance** between:

- Every pixel in the original grid
    
- Every pixel in the warped image
    

🧮 This yields:

$$
\text{dist}[b, h, w, h', w'] = \|\mathbf{p}_{hw} - \mathbf{p}'_{h'w'}\|
$$

👁️ You're building a **5D volume** where each cell tells:

> “If I warp pixel (h,w), how far does it land from (h',w')?”

⚠️ This is memory-intensive: shape = `(B, Hc, Wc, Hc, Wc)`

---

```python
s = tf.to_float(tf.less_equal(cell_distances, config['grid_size'] - 0.5))
```

✅ Now you **threshold**: mark as a **positive match** if the warped descriptor is close enough (≤7.5 pixels for `grid_size=8`)

🧮 Binary mask `s[b, h, w, h', w'] ∈ {0, 1}`

👁️ You’re saying:

> “Pixel (h,w) in image A matches (h′,w′) in warped image if they're close after warping.”

---

### 🧬 4. Descriptor normalization + dot products

```python
descriptors = tf.reshape(descriptors, [B, Hc, Wc, 1, 1, D])
descriptors = tf.nn.l2_normalize(descriptors, -1)
warped_descriptors = tf.reshape(warped_descriptors, [B, 1, 1, Hc, Wc, D])
warped_descriptors = tf.nn.l2_normalize(warped_descriptors, -1)
dot_product_desc = tf.reduce_sum(descriptors * warped_descriptors, -1)
```

✅ Compute the **dot product** between **every descriptor** in the image and every **warped descriptor**

🧮

$$
\text{dot}[b,h,w,h',w'] = \langle \hat{d}_{hw}, \hat{d}'_{h'w'} \rangle
$$

Where each descriptor is normalized:

$$
\hat{d} = \frac{d}{\|d\|_2} \Rightarrow \text{dot product} = \text{cosine similarity}
$$

👁️ This gives you a **similarity matrix** between all pixel pairs.

---

```python
dot_product_desc = tf.nn.relu(dot_product_desc)
```

✅ Truncate negative similarities. This prevents the model from pushing **negatives too hard**.

---

```python
dot_product_desc = tf.nn.l2_normalize(tf.reshape(dot_product_desc, [B, Hc, Wc, Hc * Wc]), 3)
dot_product_desc = tf.reshape(tf.nn.l2_normalize(tf.reshape(dot_product_desc, [B, Hc * Wc, Hc, Wc]), 1), [B, Hc, Wc, Hc, Wc])
```

✅ This does **row-wise and column-wise soft normalization**. A trick to stabilize the training.

🧮 For each descriptor, normalize similarity distribution across other locations.

👁️ Think of it like **normalizing the heatmap** over pixel matches.

---

### 🔥 5. Loss: contrastive margin

```python
positive_dist = tf.maximum(0., config['positive_margin'] - dot_product_desc)
negative_dist = tf.maximum(0., dot_product_desc - config['negative_margin'])
loss = config['lambda_d'] * s * positive_dist + (1 - s) * negative_dist
```

✅ Standard **contrastive loss**:

- For positives: want dot product **≥ positive margin**
    
- For negatives: want dot product **≤ negative margin**
    

🧮

$$
\text{Loss}_{pos} = \max(0, m^+ - \text{sim}) \quad \text{Loss}_{neg} = \max(0, \text{sim} - m^-)
$$

---

### 🛡️ 6. Valid mask for border pixels

```python
valid_mask = tf.ones(...) if valid_mask is None else valid_mask
valid_mask = tf.space_to_depth(valid_mask, grid_size)
valid_mask = tf.reduce_prod(valid_mask, axis=3)
valid_mask = tf.reshape(valid_mask, [B, 1, 1, Hc, Wc])
```

✅ Avoids matches where the original image had padding/border artifacts.

- `space_to_depth` maps image-level mask into cell-level mask
    
- `reduce_prod` acts like an AND across pixels in a cell
    

---

### 📉 7. Final normalization and loss

```python
normalization = tf.reduce_sum(valid_mask) * tf.to_float(Hc * Wc)
loss = tf.reduce_sum(valid_mask * loss) / normalization
```

✅ Normalize over all valid descriptor comparisons to get a **mean loss per image**

---

### 🧾 Summary Diagram

```text
 Image A                 Homography             Image B
   ↓                                             ↓
Descriptors A        → Warp coords         → Descriptors B
   ↓                      ↓                      ↓
Center grid coords   Warped grid coords    Cosine similarities (N²)
   ↓                      ↓                      ↓
       <——— Filter matches by L2 distance threshold ———>
   ↓                      ↓                      ↓
         Contrastive loss: pull positives together,
                            push negatives apart
```

---

### 🧠 Insights

| Component               | Purpose                                | Cost / Memory |
| ----------------------- | -------------------------------------- | ------------- |
| All-to-all dot products | Matching every pixel-pair              | ❌ Very high   |
| Warped coord distance   | Estimate ground-truth matches          | ❌ High        |
| ReLU + L2 normalization | Stabilize similarity distributions     | ✅ Efficient   |
| Valid mask              | Avoid training on invalid regions      | ✅ Efficient   |
| Contrastive loss        | Supervise descriptors discriminatively | ✅ Standard    |

### 🔍 PART 2 — Memory issue at `cell_distances = coor_cells - warped_coor_cells`

### 🔥 Why it explodes

You're doing:

```python
coor_cells = torch.Size([1, 1, 1, Hc, Wc, 2])
warped_coor_cells = torch.Size([1, Hc, Wc, 1, 1, 2])
cell_distances = coor_cells - warped_coor_cells
```

Which **broadcasts to:**

```
(1, Hc, Wc, Hc, Wc, 2)  → ≈ (20160×20160×2)
```

→ This is an **outer subtraction** for every cell vs every warped cell → OOM at 20GB+.

---

### ✅ Efficient fix: vectorised subtraction with flattening

Instead of outer subtraction, do:

```python
# flatten
coor_cells_flat = coor_cells.view(-1, 2)              # [N, 2]
warped_coor_cells_flat = warped_coor_cells.view(-1, 2)  # [N, 2]

# [N, 2] → [N, N, 2] difference
delta = coor_cells_flat[:, None, :] - warped_coor_cells_flat[None, :, :]  # [N, N, 2]
cell_distances = torch.norm(delta, dim=-1)  # [N, N]
```

→ That’s ~2×N² floats instead of ~N²×2×Hc×Wc. Memory drops from **20GB+ to ~1GB**.

You can also limit it by a radius (see LoFTR-style local windowing later if needed).

---

### 💡 Additional suggestion

If you're only interested in **matches near the warped coordinates**, you can avoid all-pairs with:

```python
# compute distance only near the diagonal
R = 5  # radius
for i in range(N):
    j_start = max(0, i-R)
    j_end   = min(N, i+R)
    # compute distance[i, j_start:j_end]
```

---

| Task                               | Fix                                                 | Memory reduction     |
| ---------------------------------- | --------------------------------------------------- | -------------------- |
| All-pairs dot product              | Replace outer product with `torch.bmm` on flattened | ❌ 416 GB → ✅ 1.6 GB  |
| Coordinate distance calc           | Flatten & vectorise outer subtraction               | ❌ 20+ GB → ✅ 1 GB    |
