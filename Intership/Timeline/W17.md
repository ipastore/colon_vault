### SuperPoint Training File Identification

Looking at the SuperPoint repository structure, I notice that my previous command suggestions were incorrect. This implementation uses a different file structure than what I initially described.

#### Correct Training Command

The main entry point for training in this repository is experiment.py, not a standalone `train.py` file. The correct command would be:

```bash
python -m superpoint.experiment train configs/superpoint_colonoscopy.yaml experiment_name
```

Where:
- `experiment_name` is a name you choose for your training run
- You can add `--eval` to evaluate after training
- Add `--pretrained_model MODEL_NAME` to start from a pretrained model

#### Correct Evaluation Command

Similarly, for evaluation:

```bash
python -m superpoint.experiment evaluate configs/superpoint_colonoscopy.yaml experiment_name
```

#### Understanding the Implementation

This SuperPoint implementation uses TensorFlow (not PyTorch as I incorrectly suggested) and has:

1. A custom experiment runner (experiment.py)
2. A model class hierarchy with `BaseModel` as parent class
3. Command-line parsing that expects subcommands: `train`, `evaluate`, or `predict`

When looking at experiment.py, you can see the main function that parses arguments and dispatches to the appropriate handler (`_cli_train`, `_cli_eval`, etc.).

#### Next Steps

1. Create your configuration file at `configs/superpoint_colonoscopy.yaml`
2. Run the training with the correct command above
3. Monitor the results in the experiment output directory
